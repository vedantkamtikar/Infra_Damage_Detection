# Infra_Damage_Detection — Project Notes

## Project Goal
Design and implement an autonomous multirotor drone system in a high-fidelity simulation environment that performs vision-based navigation and inspection tasks using real-time object detection. The system operates entirely in simulation using Unreal Engine and AirSim, with Python controlling perception, decision-making, and flight behavior.

## Key Decisions

### Model
- **YOLO11m** (Ultralytics YOLO11, medium variant) and **YOLO26m**
- Chosen over v8/v9/v10 for better accuracy, same Ultralytics API, more recent for academic reporting
- We are now using YOLO26 for run 9 and onwards.

### Dataset Split
```
80% Training
10% Validation
10% Testing
```

## Perception Pipeline (How Inference Works)
1. Capture RGB frame from drone camera via AirSim
2. Pass frame to YOLO26m
3. YOLO detects crack regions and draws bounding boxes
4. Extract class, confidence score, bbox coordinates
5. Log detection to database (SQLite) with drone's current NED/GPS coordinates from AirSim
6. Display annotated frame in real time

---

## Navigation Strategy

### Phase 1 — Predefined Waypoint Navigation
- Follow sequence of 3D coordinates
- Maintain stable altitude
- Execute patrol/inspection routes

### Phase 2 — Vision-Guided Behavior
- Modify trajectory based on YOLO detections
- Perform inspection maneuvers (e.g., hover, circle structure)
- Trigger location logging when damage detected
