# Infra_Damage_Detection — Project Notes

## Project Goal
Design and implement an autonomous multirotor drone system in a high-fidelity simulation environment that performs vision-based navigation and inspection tasks using real-time object detection. The system operates entirely in simulation using Unreal Engine and AirSim, with Python controlling perception, decision-making, and flight behavior.

## Dataset
- **Name:** SDNET2018 (saved as `SDNET` in datasets folder)
- **Source:** Kaggle
- **Structure:**
```
SDNET/
├── Decks/
│   ├── Cracked/
│   └── Non-cracked/
├── Pavements/
│   ├── Cracked/
│   └── Non-cracked/
└── Walls/
    ├── Cracked/
    └── Non-cracked/
```
- **Description:** Concrete surface crack images tightly cropped around the surface area. Binary per folder — cracked or non-cracked.

---

## Key Decisions

### Model
- **YOLO11m** (Ultralytics YOLO11, medium variant)
- Chosen over v8/v9/v10 for better accuracy, same Ultralytics API, more recent for academic reporting

### Detection Approach — Option B (Whole Image Bounding Boxes)
After evaluating three options:

- **Option A** — Classification + Tiling: Train as classifier, tile live frames, draw boxes on cracked patches. Rejected because grid boxes look unnatural in realistic scenes.
- **Option B** — Object Detection with whole-image bounding boxes: Convert SDNET2018 to YOLO detection format using full-image bbox for cracked images. **CHOSEN.**
- **Option C** — Proper annotation with tight bounding boxes via Roboflow. Rejected because it abandons SDNET2018.

**Why Option B:**
- SDNET images are already tightly cropped around the surface so whole-image bbox is close to the actual crack location
- Produces proper YOLO detection output with boxes on live feed
- Simple and clean pipeline
- Academically defensible — used in published crack detection papers

### Classes (4 total)
```
0: deck_crack
1: pavement_crack
2: wall_crack
3: no_crack
```

### Dataset Split
```
80% Training
10% Validation
10% Testing
```

## Full Pipeline
```
SDNET2018 Images
      ↓
prepare_dataset.py  →  converts to YOLO format, generates whole-image bboxes, splits data
      ↓
sdnet.yaml  →  tells YOLO where data is and defines classes
      ↓
train.py  →  fine-tunes YOLO11m on prepared dataset
      ↓
best.pt  →  saved trained model weights
      ↓
detect.py  →  runs inference on images/video/drone feed, draws bounding boxes
      ↓
damage_log.csv  →  logs timestamp, drone position, class, confidence, bbox coords
```

---

## Perception Pipeline (How Inference Works)
1. Capture RGB frame from drone camera via AirSim
2. Pass frame to YOLO11m
3. YOLO detects crack regions and draws bounding boxes
4. Extract class, confidence score, bbox coordinates
5. Log detection to CSV with drone's current NED/GPS coordinates from AirSim
6. Display annotated frame in real time

---

## Navigation Strategy

### Phase 1 — Predefined Waypoint Navigation
- Follow sequence of 3D coordinates
- Maintain stable altitude
- Execute patrol/inspection routes

### Phase 2 — Vision-Guided Behavior
- Modify trajectory based on YOLO detections
- Perform inspection maneuvers (e.g., hover, circle structure)
- Trigger location logging when damage detected
